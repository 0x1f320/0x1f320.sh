## What Is an Optimistic Update

The typical flow: press a button, send a request, show a spinner until the server responds. But most requests succeed. So **if we assume success and update the UI immediately**, the perceived speed improves dramatically.

## React 19's useOptimistic

```tsx
import { useOptimistic } from "react";

function TodoList({ todos, addTodo }) {
  const [optimisticTodos, setOptimisticTodos] = useOptimistic(
    todos,
    (state, newTodo) => [...state, { ...newTodo, pending: true }],
  );

  async function handleAdd(text) {
    const newTodo = { id: crypto.randomUUID(), text };
    setOptimisticTodos(newTodo);
    await addTodo(newTodo); // Server Action
  }

  return (
    <ul>
      {optimisticTodos.map((todo) => (
        <li key={todo.id} style={{ opacity: todo.pending ? 0.5 : 1 }}>
          {todo.text}
        </li>
      ))}
    </ul>
  );
}
```

Items with `pending: true` render at half opacity. Once the server responds, they're replaced with real data.

## Caveats

- You need rollback logic for failures
- Not ideal for order-sensitive operations
- Overusing optimistic updates on flaky networks can confuse users

> Fast UI doesn't come from a fast server — it comes from not waiting.
