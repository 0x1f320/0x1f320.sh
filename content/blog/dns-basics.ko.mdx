## DNS란

도메인 이름을 IP 주소로 변환하는 시스템입니다. `example.com` → `93.184.216.34`.

## 레코드 타입

### A 레코드

도메인 → IPv4 주소

```
example.com.  A  93.184.216.34
```

### CNAME

도메인 → 다른 도메인 (별칭)

```
www.example.com.  CNAME  example.com.
```

### AAAA

도메인 → IPv6 주소

### MX

메일 서버 지정

### TXT

도메인 소유권 인증, SPF, DKIM 등

## TTL

Time To Live. 캐시 유효 시간(초).

- 평소: `3600` (1시간)
- 마이그레이션 전: `60`으로 낮춤 → 전환 → 다시 올림

## 흔한 실수

- CNAME을 루트 도메인에 설정 — 표준 위반 (일부 DNS는 ALIAS로 우회)
- TTL을 낮추지 않고 DNS 전환 — 구 IP로 트래픽이 계속 감
- www와 루트 도메인을 둘 다 설정하지 않음

> DNS는 인프라의 첫 관문입니다. 여기서 막히면 아무것도 작동하지 않습니다.
