## The Problem with Promise.all

```typescript
const results = await Promise.all([
  fetchUser(),
  fetchOrders(),
  fetchNotifications(),
]);
```

If one fails, they all fail. There's no reason to hide user info just because notification fetch failed.

## Promise.allSettled

```typescript
const results = await Promise.allSettled([
  fetchUser(),
  fetchOrders(),
  fetchNotifications(),
]);

for (const result of results) {
  if (result.status === "fulfilled") {
    // result.value
  } else {
    // result.reason — handle each error individually
  }
}
```

## When to Use What

| Scenario | Choice |
|----------|--------|
| All results required | `Promise.all` |
| Partial failure OK | `Promise.allSettled` |
| Just one success needed | `Promise.any` |
| Fastest result wins | `Promise.race` |

> Decide how to handle failure first, then pick the right pattern.
