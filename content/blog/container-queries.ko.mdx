## 미디어 쿼리의 한계

미디어 쿼리는 뷰포트 기준입니다. 사이드바가 있으면 같은 뷰포트에서도 컴포넌트가 쓸 수 있는 너비가 달라집니다.

## Container Query

```css
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}
```

컴포넌트가 자신의 컨테이너 크기에 반응합니다. 어디에 놓이든 동일하게 작동합니다.

## 실전 사용처

- 대시보드의 위젯 카드
- CMS에서 다양한 레이아웃에 배치되는 컴포넌트
- 사이드바 토글 시 메인 영역 레이아웃 변경

## 대체할 수 있나?

완전히는 아닙니다. 글로벌 레이아웃 변경(네비게이션 숨기기 등)은 여전히 미디어 쿼리가 필요합니다. 하지만 컴포넌트 레벨에서는 container query가 더 적합합니다.

> 뷰포트가 아니라 컨텍스트에 반응하세요.
