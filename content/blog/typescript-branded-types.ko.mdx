## 문제

대부분의 코드베이스에서 ID는 `string`입니다.

```typescript
function getUser(userId: string) { /* ... */ }
function getOrder(orderId: string) { /* ... */ }

const orderId = "order_abc123";
getUser(orderId); // 컴파일 에러 없음 — 런타임에 터짐
```

`userId`와 `orderId`가 둘 다 `string`이기 때문에 TypeScript는 이걸 구분하지 못합니다.

## Branded Type

타입에 유령 프로퍼티를 붙여서 구조적으로 호환되지 않게 만듭니다.

```typescript
type Brand<T, B extends string> = T & { readonly __brand: B };

type UserId = Brand<string, "UserId">;
type OrderId = Brand<string, "OrderId">;

function getUser(userId: UserId) { /* ... */ }

const orderId = "order_abc123" as OrderId;
getUser(orderId);
// TS2345: Argument of type 'OrderId' is not assignable to parameter of type 'UserId'.
```

이제 컴파일러가 잡아줍니다.

## 언제 쓰면 좋은가

- 여러 종류의 ID가 같은 원시 타입을 공유할 때
- 외부 API 응답을 파싱할 때 경계를 명확히 하고 싶을 때
- 비용 없이 타입 안전성만 얻고 싶을 때 — 런타임 오버헤드는 제로

> Branded type은 런타임에는 존재하지 않습니다. 순수하게 컴파일 타임 안전장치입니다.
