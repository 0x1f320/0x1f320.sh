## 레이어 캐시 원리

Docker는 각 명령어를 레이어로 저장합니다. 이전 레이어가 바뀌지 않았으면 캐시를 재사용합니다.

## 안 좋은 예

```dockerfile
COPY . .
RUN npm install
RUN npm run build
```

코드가 한 줄만 바뀌어도 `npm install`부터 다시 실행됩니다.

## 좋은 예

```dockerfile
COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm build
```

의존성 파일이 안 바뀌면 `install` 레이어를 캐시합니다.

## 추가 팁

- `.dockerignore`에 `node_modules`, `.git` 추가
- 멀티스테이지 빌드로 최종 이미지 크기 줄이기
- `--mount=type=cache`로 패키지 매니저 캐시 마운트

> 레이어 순서만 바꿔도 빌드 시간이 절반으로 줄 수 있습니다.
