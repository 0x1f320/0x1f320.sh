## Basic Concept

React Error Boundaries catch rendering errors in child components and show a fallback UI.

```tsx
<ErrorBoundary fallback={<p>Something went wrong.</p>}>
  <Dashboard />
</ErrorBoundary>
```

## Where to Place Them

### Page Level

Show an error page when the entire page breaks.

### Widget Level

Wrap each dashboard widget in its own boundary — one crash doesn't take down the rest.

### Route Level

In Next.js, `error.tsx` serves this role.

## Caveats

- Doesn't catch event handler errors — use `try/catch`
- Doesn't catch async errors — combine with Suspense
- Connect to error reporting (Sentry, etc.)

> The goal of a boundary isn't to eliminate errors — it's to limit the blast radius.
