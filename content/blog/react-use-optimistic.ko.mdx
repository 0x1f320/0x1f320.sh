## Optimistic Update란

버튼을 누르면 서버에 요청을 보내고, 응답이 올 때까지 로딩을 보여주는 게 일반적입니다. 하지만 대부분의 요청은 성공하죠. 그렇다면 **성공한다고 가정하고 UI를 먼저 바꾸면** 체감 속도가 훨씬 빨라집니다.

## React 19의 useOptimistic

```tsx
import { useOptimistic } from "react";

function TodoList({ todos, addTodo }) {
  const [optimisticTodos, setOptimisticTodos] = useOptimistic(
    todos,
    (state, newTodo) => [...state, { ...newTodo, pending: true }],
  );

  async function handleAdd(text) {
    const newTodo = { id: crypto.randomUUID(), text };
    setOptimisticTodos(newTodo);
    await addTodo(newTodo); // Server Action
  }

  return (
    <ul>
      {optimisticTodos.map((todo) => (
        <li key={todo.id} style={{ opacity: todo.pending ? 0.5 : 1 }}>
          {todo.text}
        </li>
      ))}
    </ul>
  );
}
```

`pending: true`인 항목은 반투명하게 보여주고, 서버 응답이 오면 실제 데이터로 교체됩니다.

## 주의할 점

- 실패 시 롤백 로직이 필요합니다
- 순서가 중요한 작업에는 적합하지 않을 수 있습니다
- 네트워크가 불안정한 환경에서는 낙관적 업데이트를 남용하면 혼란을 줄 수 있습니다

> 빠른 UI는 빠른 서버에서 오는 게 아니라, 기다리지 않는 것에서 옵니다.
