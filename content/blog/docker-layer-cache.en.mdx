## How Layer Cache Works

Docker stores each instruction as a layer. If a previous layer hasn't changed, it reuses the cache.

## Bad Example

```dockerfile
COPY . .
RUN npm install
RUN npm run build
```

One line of code changes and `npm install` runs again from scratch.

## Good Example

```dockerfile
COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm build
```

If dependency files haven't changed, the `install` layer is cached.

## Extra Tips

- Add `node_modules` and `.git` to `.dockerignore`
- Use multi-stage builds to reduce final image size
- Mount package manager cache with `--mount=type=cache`

> Just reordering layers can cut build time in half.
