## 기본 구조

```sql
SELECT
  name,
  department,
  salary,
  AVG(salary) OVER (PARTITION BY department) AS dept_avg
FROM employees;
```

`GROUP BY` 없이 각 행에 집계값을 붙일 수 있습니다.

## 자주 쓰는 함수

### ROW_NUMBER — 순번

```sql
ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC)
```

### RANK — 순위 (동점 허용)

```sql
RANK() OVER (ORDER BY score DESC)
```

### LAG / LEAD — 이전/다음 행

```sql
LAG(salary, 1) OVER (ORDER BY hire_date)
```

### SUM — 누적합

```sql
SUM(revenue) OVER (ORDER BY month ROWS UNBOUNDED PRECEDING)
```

## 성능 팁

- `PARTITION BY`에 인덱스가 있으면 빠릅니다
- `ROWS BETWEEN`으로 윈도우 범위를 제한하면 메모리 사용량이 줄어듭니다
- 서브쿼리보다 거의 항상 window function이 빠릅니다
