## The Test Pyramid Trap

We're told to write lots of unit tests, but sometimes 3 integration tests catch more bugs than 100 unit tests.

## The Right Questions

Ask these when writing a test:

1. **What bug would reach production without this test?** — If there's no answer, don't write it
2. **Does this test block refactoring?** — Tests coupled to implementation are debt
3. **Is a failure a meaningful signal?** — Tests that always break or never break are useless

## Practical Approach

- **Business logic**: unit tests
- **API endpoints**: integration tests
- **User flows**: E2E tests (critical paths only)
- **UI components**: Storybook over snapshots

> Hitting coverage numbers is easy. Writing meaningful tests is hard.
