## 왜 경계가 중요한가

모노레포의 장점은 코드 공유입니다. 하지만 경계 없이 공유하면 모든 것이 모든 것에 의존하는 Big Ball of Mud가 됩니다.

## 패키지를 나누는 기준

1. **배포 단위가 다를 때** — 웹앱과 모바일 앱은 별도 패키지
2. **팀이 다를 때** — 소유권이 명확해야 변경 속도가 올라감
3. **변경 빈도가 극단적으로 다를 때** — 자주 바뀌는 UI와 거의 안 바뀌는 프로토콜 정의

## 의존성 방향

```
app → feature → shared → core
```

역방향 의존은 금지. lint 규칙으로 강제하세요.

```jsonc
// .eslintrc (eslint-plugin-boundaries)
{
  "boundaries/element-types": [
    { "from": "core", "disallow": ["feature", "app"] }
  ]
}
```

> 모노레포는 경계를 없애는 게 아니라, 경계를 저렴하게 만드는 겁니다.
