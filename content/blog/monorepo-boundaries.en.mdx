## Why Boundaries Matter

The advantage of a monorepo is code sharing. But sharing without boundaries turns everything into a Big Ball of Mud where everything depends on everything.

## When to Split

1. **Different deploy targets** — web app and mobile app should be separate packages
2. **Different teams** — clear ownership speeds up changes
3. **Wildly different change frequency** — frequently changing UI vs. rarely changing protocol definitions

## Dependency Direction

```
app → feature → shared → core
```

Reverse dependencies are banned. Enforce with lint rules.

```jsonc
// .eslintrc (eslint-plugin-boundaries)
{
  "boundaries/element-types": [
    { "from": "core", "disallow": ["feature", "app"] }
  ]
}
```

> A monorepo doesn't remove boundaries — it makes them cheap.
