## What Is DNS

A system that translates domain names to IP addresses. `example.com` → `93.184.216.34`.

## Record Types

### A Record

Domain → IPv4 address

```
example.com.  A  93.184.216.34
```

### CNAME

Domain → another domain (alias)

```
www.example.com.  CNAME  example.com.
```

### AAAA

Domain → IPv6 address

### MX

Mail server specification

### TXT

Domain ownership verification, SPF, DKIM, etc.

## TTL

Time To Live. Cache validity in seconds.

- Normal: `3600` (1 hour)
- Before migration: lower to `60` → switch → raise again

## Common Mistakes

- Setting CNAME on root domain — violates the standard (some DNS providers work around with ALIAS)
- Switching DNS without lowering TTL — traffic keeps going to old IP
- Not configuring both www and root domain

> DNS is the front gate of infrastructure. If it's blocked, nothing works.
