## Limitations of Media Queries

Media queries are viewport-based. If there's a sidebar, the available width for a component differs even at the same viewport size.

## Container Queries

```css
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}
```

Components respond to their container's size. They work the same regardless of placement.

## Real-World Use Cases

- Dashboard widget cards
- CMS components placed in varying layouts
- Main area layout changes when a sidebar toggles

## Can They Replace Media Queries?

Not entirely. Global layout changes (hiding navigation, etc.) still need media queries. But at the component level, container queries are a better fit.

> Respond to context, not viewport.
