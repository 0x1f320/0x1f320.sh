## Why I Switched

- 5 config files just for ESLint + Prettier
- Plugin version conflicts
- 30 seconds for lint + format in CI

## Biome

Linter and formatter in a single binary. Written in Rust, so it's fast.

```bash
pnpm add -D @biomejs/biome
pnpm biome init
```

## Migration

```bash
pnpm biome migrate eslint --write
pnpm biome migrate prettier --write
```

Auto-converts most existing rules. Anything it can't handle goes manually into `biome.json`.

## The Difference

- CI lint time: 30s → 2s
- Config files: 5 → 1 (`biome.json`)
- Format on save: effectively instant

## Downsides

- Can't use the entire ESLint plugin ecosystem
- Some rules are still unimplemented

> It doesn't have to be perfect. 95% of the rules running 10x faster is a net win.
