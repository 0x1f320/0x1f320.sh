## 기본 개념

React의 Error Boundary는 자식 컴포넌트에서 발생한 렌더링 에러를 잡아서 fallback UI를 보여줍니다.

```tsx
<ErrorBoundary fallback={<p>문제가 발생했습니다.</p>}>
  <Dashboard />
</ErrorBoundary>
```

## 어디에 배치할까

### 페이지 레벨

전체 페이지가 깨지면 에러 페이지를 보여줍니다.

### 위젯 레벨

대시보드의 각 위젯을 개별 boundary로 감싸면, 하나가 죽어도 나머지는 살아있습니다.

### 라우트 레벨

Next.js에서는 `error.tsx`가 이 역할을 합니다.

## 주의사항

- 이벤트 핸들러의 에러는 잡지 못합니다 — `try/catch` 사용
- 비동기 에러도 잡지 못합니다 — Suspense와 조합
- 에러 리포팅 서비스(Sentry 등)와 연동하세요

> Boundary의 목적은 에러를 없애는 게 아니라, 피해 범위를 줄이는 겁니다.
